Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

To begin answering this question, we must first know what semantic tags mean. According to the article "Semantic HTML", semantic means "relating to meaning". When we apply this in HTML, this would mean that we are basing the strucutre of HTML on it's meaning, not appearance.

When we use a non semantic elements it's harder for us to know what it's representing. Basically semantic elements allow us to know the purpose and or meaning of the code we are trying to achieve without using the HTML tag.

What are two (or more) benefits of using semantic tags?

Now that we know what semantics in HTML are, we can see how it can have many benefits in coding. According to the article "A Detailed Guide on HTML Semantics", there are multiple benefits including structure, accessibility, compatibility, etc.

One of those benefits is accessibility.Semantic HTML allows those with disabilities to have a friendlier user experience, as well as a clear strucutre.

A second benefit would be browser compatibility. With new tehcnologies, semantic elements allow for adaptability with future technology.
